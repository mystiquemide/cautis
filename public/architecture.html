Cautis Architecture

Guardian Protocol for AI Agents on Casper Network

Agent Registration Flow

  • 1. Operator stakes CSPR via frontend
  • 2. Guardian Contract deployed on Casper Testnet
  • 3. Agent registered with daily limit + stake
  • 4. Status: active, monitored by Guardian AI

Monitoring & Freeze Flow

  • 1. POST /api/monitor with transaction data
  • 2. Guardian agent analyzes via Claude or quickCheck
  • 3. If anomalous: freeze_agent() called on-chain
  • 4. Alert created, funds secured, operator notified

Tech Stack

  • Smart Contracts: Odra (Rust) on Casper
  • Guardian AI: Anthropic Claude API
  • Backend: Next.js 16 API Routes
  • Chain Access: CSPR.cloud + x402
  • Frontend: Next.js + Tailwind CSS v4
